# 自販機システム テスト仕様書

## 1. 目的
本書は [docs/ai_design.md](docs/ai_design.md) と [docs/ai_detailed_design.md](docs/ai_detailed_design.md) を基に、
自販機システムの機能および関数が設計どおりに動作することを確認するテスト仕様を定義する。

## 2. 対象範囲
- 対象プログラム: `src/main.c`
- 対象機能（基本設計）:
  - F01 商品一覧表示
  - F02 投入金額入力
  - F03 商品購入
  - F04 お釣り管理
  - F05 購入履歴管理
  - F06 エラー処理
  - F07 購入履歴表示（拡張）
  - F08 在庫補充（拡張）
- 対象関数（詳細設計）:
  - showProductList, inputMoney, buyProduct, calcChange, savePurchase
  - showMenu, main, getValidInt, showPurchaseHistory, restockProduct

## 3. テストレベル
- 単体テスト: 各関数の正常系・異常系・境界値
- 結合テスト: 購入フローの一連動作（一覧表示→選択→投入→購入→履歴）
- システムテスト: メニュー操作での実利用シナリオ

## 4. テスト環境
- OS: Windows
- 実行環境: VSCode ターミナル
- ビルドコマンド:
  - `gcc -finput-charset=utf-8 -fexec-charset=cp932 src/main.c -o app.exe`
- 実行コマンド:
  - `./app.exe`
- テストデータ:
  - 商品データ（main.c の初期配列）
  - 履歴ファイル: `purchase_history.csv`

## 5. 受入基準
- 全必須テストケース（優先度 High）が PASS であること。
- 中優先度（Medium）は 90%以上 PASS であること。
- Fail がある場合、原因と暫定対策を記録し再試験すること。

## 11. 注意事項
- `showMenu`, `getValidInt`, `showPurchaseHistory`, `restockProduct` は実装状況により N/A となる可能性がある。
- N/A の場合は未実装として記録し、実装完了後に再試験する。
- 文字コード差異により履歴CSVの日本語表示が乱れる場合、実行環境とファイルエンコーディング（UTF-8/CP932）を合わせて確認する。
