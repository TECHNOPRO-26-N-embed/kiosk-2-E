# 自販機システム 詳細設計書（関数詳細）

## 1. 目的
本書は、基本設計書に記載された各関数について、実装時に迷わない粒度で以下を定義する。
- アルゴリズム
- 入出力
- エラー処理

対象は [kiosk-2-e/docs/ai_design.md](kiosk-2-e/docs/ai_design.md) の「5. 実装すべき関数一覧と役割」に記載された関数とする。

## 2. 前提と共通仕様
- 商品情報は CSV から読み込む、またはメモリ上配列で保持する。
- 金額は整数（円）で扱う。
- 入力は標準入力、表示は標準出力を使用する。
- 入力エラー時は即時終了せず、再入力を促す。
- ファイル I/O 失敗時はユーザーへ通知し、処理継続可否を関数ごとに定義する。

## 3. 関数詳細

### 3.1 void showProductList()
#### 目的
商品ID、商品名、価格、在庫数を一覧表示する。

#### 入出力
- 入力
  - なし（内部で商品データ配列または商品CSV読込結果を参照）
- 出力
  - 画面表示（商品一覧）
  - 戻り値なし

#### アルゴリズム
1. 商品データ件数を取得する。
2. 見出し（ID/商品名/価格/在庫）を表示する。
3. 先頭から末尾まで商品を1件ずつ処理する。
4. 在庫数が 0 の場合は「売り切れ」を表示する。
5. それ以外は在庫数をそのまま表示する。
6. 一覧出力後に操作案内（商品ID入力、終了方法など）を表示する。

#### エラー処理
- 商品データ未読込/件数0:
  - 「商品データがありません」と表示。
  - 呼び出し元へ戻る。
- 価格または在庫が負値:
  - 異常データとしてログ出力。
  - 当該行は「データ異常」と表示し、処理継続。

### 3.2 int inputMoney()
#### 目的
ユーザーから投入金額を受け取り、妥当な整数を返す。

#### 入出力
- 入力
  - 標準入力からの金額文字列
- 出力
  - 妥当な投入金額（int）

#### アルゴリズム
1. 「投入金額を入力してください」と表示する。
2. 1行入力を受け取る。
3. 空入力をチェックする。
4. 数字のみで構成されているか検証する。
5. 整数に変換し、0以上か判定する。
6. 上限値（例: 10000円）を超える場合は再入力。
7. 妥当な値を返す。

#### エラー処理
- 空入力/非数値入力:
  - エラーメッセージ表示後、再入力。
- 負値/上限超過:
  - エラーメッセージ表示後、再入力。
- 変換オーバーフロー:
  - エラーメッセージ表示後、再入力。

### 3.3 void buyProduct()
#### 目的
商品購入の一連処理（商品選択、在庫確認、金額確認、在庫更新、履歴保存）を実行する。

#### 入出力
- 入力
  - 商品ID（ユーザー入力）
  - 投入金額（inputMoney() の戻り値）
  - 商品データ（内部配列/CSV読込結果）
- 出力
  - 画面表示（購入結果、お釣り、エラー）
  - 必要に応じて履歴CSV追記

#### アルゴリズム
1. showProductList() で商品一覧を表示する。
2. 商品IDを入力させる（getValidInt() 利用）。
3. 商品IDの存在チェックを行う。
4. 在庫数を確認する。
5. 在庫ありなら inputMoney() で投入金額を受け取る。
6. 金額不足判定（money < price）を行う。
7. 十分なら商品排出メッセージを表示する。
8. 在庫を1減算する。
9. calcChange(money, price) でお釣りを計算し表示する。
10. savePurchase(productName, price) を呼び履歴保存する。
11. 処理結果を表示して終了。

#### エラー処理
- 不正商品ID:
  - 「存在しない商品IDです」を表示し、再選択。
- 売り切れ（在庫0）:
  - 「売り切れです」を表示し、購入処理を中断。
- 金額不足:
  - 不足額（price - money）を表示し、購入中断または再投入を案内。
- 在庫更新失敗（保存型実装時）:
  - エラー表示、ログ記録、購入確定を取り消し。
- 履歴保存失敗:
  - 購入は成立扱いとし、履歴保存失敗を警告表示。

### 3.4 int calcChange(int money, int price)
#### 目的
投入金額と価格からお釣りを計算する。

#### 入出力
- 入力
  - money: 投入金額
  - price: 商品価格
- 出力
  - お釣り額（int）

#### アルゴリズム
1. money と price の妥当性（0以上）を確認。
2. money >= price の場合、money - price を返す。
3. money < price の場合、-1 を返す（不足のシグナル）。

#### エラー処理
- money < 0 または price < 0:
  - 異常引数として -1 を返す。
- 整数範囲外の異常（実装時に想定される場合）:
  - 例外的に 0 を返さず -1 で異常扱い。

### 3.5 void savePurchase(const char* productName, int price)
#### 目的
購入履歴（商品名、価格、日時）をCSVへ追記する。

#### 入出力
- 入力
  - productName: 商品名
  - price: 価格
- 出力
  - purchase_history.csv への追記
  - 戻り値なし

#### アルゴリズム
1. 現在日時を取得する。
2. 履歴ファイルを追記モードで開く。
3. 1レコードをCSV形式で書き込む。
4. 書き込み結果を確認する。
5. ファイルを閉じる。

#### エラー処理
- productName が NULL または空:
  - 「履歴保存対象が不正」として保存中止。
- ファイルオープン失敗:
  - 画面警告とログ記録。
- 書き込み失敗/クローズ失敗:
  - 警告表示し、再試行は行わず呼び出し元へ戻る。

### 3.6 void showMenu()
#### 目的
メニューを表示し、選択可能な操作をユーザーに提示する。

#### 入出力
- 入力
  - なし（表示のみ）
- 出力
  - メニュー表示（例: 1.購入 2.履歴表示 3.在庫補充 9.終了）

#### アルゴリズム
1. 画面にメニュー見出しを表示。
2. 選択肢を行単位で表示。
3. 入力方法と有効範囲を表示。

#### エラー処理
- 表示処理自体の致命エラーは基本想定しない。
- 異常な表示データ検知時は安全な固定文言にフォールバック。

### 3.7 int main()
#### 目的
システム全体の起動、初期化、メインループ、終了処理を管理する。

#### 入出力
- 入力
  - メニュー選択値（ユーザー入力）
- 出力
  - 各機能呼び出し、終了コード（int）

#### アルゴリズム
1. 商品データ/設定データを初期化する。
2. 初期化失敗ならエラー表示して終了コード 1 を返す。
3. 終了指示が出るまでループする。
4. showMenu() を表示する。
5. getValidInt() でメニュー番号を取得する。
6. 選択に応じて buyProduct()/showPurchaseHistory()/restockProduct() などを呼ぶ。
7. 9 が入力されたらループ終了。
8. 必要な後処理（ファイルクローズ等）を実施し、0 を返す。

#### エラー処理
- 不正メニュー番号:
  - エラーメッセージ表示後、ループ継続。
- 初期化失敗:
  - 詳細を表示し異常終了。
- 実行時予期せぬ分岐:
  - ログ記録して安全にメニューへ戻す。

### 3.8 int getValidInt(const char* prompt)
#### 目的
整数入力を安全に受け取る共通関数。

#### 入出力
- 入力
  - prompt: 入力促進メッセージ
  - 標準入力の文字列
- 出力
  - 妥当な整数値

#### アルゴリズム
1. prompt を表示する。
2. 入力文字列を取得する。
3. 前後空白を除去する。
4. 符号/数字以外の文字有無を検証する。
5. 整数変換し、範囲を確認する。
6. 妥当値なら返却、異常なら再入力。

#### エラー処理
- prompt が NULL:
  - 既定メッセージを使用し継続。
- 入力ストリームエラー（EOF 含む）:
  - -1 を返却し呼び出し元に終了判断を委譲。
- 変換失敗/範囲外:
  - エラーメッセージ表示後、再入力。

### 3.9 void showPurchaseHistory()
#### 目的
購入履歴CSVを読み込み、一覧表示する。

#### 入出力
- 入力
  - 購入履歴CSVファイル
- 出力
  - 履歴一覧の画面表示

#### アルゴリズム
1. 履歴ファイルを読込モードで開く。
2. ヘッダ行を読み込む（必要なら表示）。
3. 各レコードを1行ずつ読み取り表示する。
4. 件数を集計し、最後に総件数を表示。
5. ファイルを閉じる。

#### エラー処理
- ファイル未存在:
  - 「履歴がありません」と表示し正常復帰。
- 読込失敗/フォーマット異常:
  - 異常行をスキップし、警告を表示。
- ファイルクローズ失敗:
  - 警告のみ表示。

### 3.10 void restockProduct(int id, int amount)
#### 目的
指定商品IDの在庫を補充する。

#### 入出力
- 入力
  - id: 商品ID
  - amount: 補充数
- 出力
  - 在庫更新結果の表示
  - 必要に応じて商品データ保存

#### アルゴリズム
1. id の存在確認を行う。
2. amount が正数か判定する。
3. 現在在庫に amount を加算する。
4. 上限在庫（例: 50）を超える場合は上限で丸める、またはエラーとする。
5. 更新後在庫を表示する。
6. 永続化が必要な実装ではデータ保存する。

#### エラー処理
- 不正ID:
  - エラー表示し更新中止。
- amount <= 0:
  - エラー表示し更新中止。
- 在庫上限超過:
  - 方針に従い補正または更新拒否。
- 保存失敗:
  - 警告表示し再試行可否を案内。

## 4. 返却値・エラーコード方針（推奨）
実装の一貫性のため、以下の戻り値方針を推奨する。
- 正常: 0 または正値
- 入力不正: -1
- ファイルI/O失敗: -2
- データ不整合: -3

void 関数で失敗を通知したい場合は、将来拡張として int 戻り値化を検討する。

## 5. テスト観点（関数単位）
- 正常系: 想定入力で期待結果が出ること。
- 異常系: 空入力、文字入力、範囲外入力、ファイル障害時の復帰動作。
- 境界値: 0円、価格と同額、在庫0、在庫上限ちょうど、ID最小/最大。
