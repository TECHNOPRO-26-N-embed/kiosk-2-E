# 自動販売機 要件定義書（実装可能範囲内）

## 顧客要望
- 商品数: 50
- 在庫: 1種類あたり50
- 季節ごとに商品入れ替えあり
- 止まらないように（安定稼働）
- 金額表示
- 飲み物だけを販売

## 必要な機能一覧（実装可能範囲内）

### 1. 商品管理機能
- 最大50種類の商品を登録可能。
- 各商品の在庫数を管理（1種類あたり最大50個）。
- 季節ごとの商品入れ替えは手動で対応（管理者用インターフェースを提供）。

### 2. 販売機能
- 飲み物のみを販売。
- 商品の金額を表示。
- 購入時に在庫数を減少。
- 売り切れ表示機能。

### 3. ログ監視機能
- ログデータを記録し、管理者が確認可能。

### 4. 安定稼働機能
- 障害発生時のログ記録。
- 障害発生の想定（ログ記録対象）:
	- 入力障害（範囲外の商品番号、数値以外の入力、空入力の連続など）。
	- 在庫整合性の障害（在庫0での購入処理、在庫のマイナス化、在庫更新失敗）。
	- データI/O障害（商品データ・在庫データ・ログファイルの読み書き失敗、ファイル破損）。
	- 金額処理の障害（未定義価格、負の価格、合計値の不整合、表示失敗）。
	- 実行時エラー（想定外分岐や参照エラー等による異常終了）。
	- 連続エラーによるサービス低下（短時間で同種エラーが多発）。

### 5. ユーザーインターフェース
- 商品選択ボタン（CLIベースで実装）。
- 金額表示（ターミナルに出力）。
- 購入完了時の通知（ターミナルにメッセージを表示）。

### 6. メンテナンス機能
- 在庫補充の通知（ターミナルに警告を表示）。
- 季節商品の入れ替え管理（手動で商品リストを更新）。

---

以上が実装可能範囲内での自動販売機の要件定義です。
