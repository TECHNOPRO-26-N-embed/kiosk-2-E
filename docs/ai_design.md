# 自販機システム 基本設計書

---

## 1. 概要

本設計書は、飲料自販機システムの基本設計をまとめたものである。主な目的は、ユーザーが商品一覧から飲料を選び、金銭投入・購入・お釣り・履歴管理まで一連の流れを安全かつ分かりやすく実現することである。

---

## 2. 機能一覧

| 機能ID | 機能名         | 概要                         |
|:------:|:---------------|:-----------------------------|
| F01    | 商品一覧表示   | 商品名・価格・在庫を表示     |
| F02    | 投入金額表示   | 投入金額の入力・表示         |
| F03    | 商品購入       | 商品選択・在庫/金額チェック  |
| F04    | お釣りの管理   | お釣り計算・排出             |
| F05    | 購入履歴管理   | 購入内容をCSV保存            |
| F06    | エラー処理     | 異常入力や不足金額の対応     |
| F07    | 購入履歴の表示 | 購入履歴CSVを読み取り表示    |
| F08    | 在庫補充       | 商品の在庫数を追加・補充     |

---

## 3. 画面遷移図

商品一覧表示 → 金銭投入 → 購入画面 → [確定] → 商品排出 → お釣り排出 → 商品一覧表示

## 4. データ設計

### 商品データのCSV形式
商品データは以下の形式でCSVファイルに定義します。

```
id,商品名,価格,在庫数
1,コーラ,120,50
2,お茶,100,30
3,水,80,20
```

- **id**: 商品ID（整数）
- **商品名**: 商品の名前（文字列）
- **価格**: 商品の価格（整数、単位: 円）
- **在庫数**: 商品の在庫数（整数）

## 5. 実装すべき関数一覧と役割

### 5.1 商品一覧表示関連
void showProductList()
  - 商品ID、商品名、価格、在庫数を一覧表示する。
  - 在庫が0の場合は「売り切れ」と表示。

### 5.2 投入金額関連
int inputMoney()
  - ユーザーから投入金額を受け取り、正の整数のみを許可。
  - 入力値を返す。

### 5.3 商品購入関連
void buyProduct()
  - 商品IDの入力、在庫・金額チェック、商品排出・在庫減少・お釣り計算まで一連の購入処理を行う。

### 5.4 お釣り管理関連
int calcChange(int money, int price)
  - 投入金額と商品の価格からお釣りを計算し返す。

### 5.5 購入履歴関連
void savePurchase(const char* productName, int price)
  - 購入した商品名・価格・日時をCSVファイルに追記。

### 5.6 メニュー・メインループ
void showMenu()
  - メニュー表示とユーザー入力受付。
int main()
  - プログラムのエントリーポイント。メインループを管理。

### 5.7 その他（エラーハンドリング等）
int getValidInt(const char* prompt)
  - 整数値の入力を安全に受け取る（数字以外や範囲外の値を弾く）。

### 5.8 拡張機能（例）

#### 購入履歴の表示
void showPurchaseHistory()
  - CSVファイルから購入履歴を読み込み、一覧表示する。

#### 在庫補充
void restockProduct(int id, int amount)
  - 指定した商品IDの在庫数をamount分増やす。

## 6. 詳細アルゴリズム（抜粋）

### 6.1 メインループ
1. 商品一覧を表示する
2. ユーザーの入力を受け取る（金銭投入）
   - 正しい値の場合: 商品購入へ
   - それ以外: 「正しい値を入力してください」と表示し再入力
3. 商品購入処理
4. 在庫・金額チェック
   - 在庫あり: 商品名・価格表示、投入金額受付
   - 在庫なし: 「売り切れ」と表示し商品選択へ戻る
5. 金額が足りていれば商品排出・お釣り計算・在庫減少
6. 購入履歴保存
7. 1に戻る

### 6.2 個別機能アルゴリズム
- 商品一覧表示: 商品データを順に表示、在庫0は「売り切れ」
- 投入金額表示: 0円→ユーザー入力→正の整数のみ許可
- 商品購入: 商品ID入力→在庫・金額チェック→排出・履歴保存
- お釣り管理: 投入金額-価格>=0なら排出
- 購入履歴: CSVファイルに追記

## 7. エラーハンドリング

| 異常系シナリオ                | 対処方針                         |
|:------------------------------|:---------------------------------|
| 数字以外の文字が入力された場合 | 再入力を促す                     |
| 金額が不足している場合         | 「金額が足りません」と表示し再入力|
| 商品番号が不正                | 「商品番号が間違っています」と表示|
| 在庫が0の場合                 | 「売り切れ」と表示し再選択        |
