## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

> メニュー表示→入力受付→処理分岐→ループの流れを記述してください。
> フローチャートまたは疑似コード（箇条書き）で記述してください。

```
1. 商品一覧を表示する
2. ユーザーの入力を受け取る(金銭投入)
   入力に応じて分岐する
   - 正しい値の場合: （商品購入の4に進む）
   - それ以外: （正しい値を入力してくださいと表示、2に戻る）
3.商品購入
4.  在庫があるか確認
5.  ある場合->6．
    ない場合->「売り切れ」と表示->1.
6.  商品名、価格を表示
8.  価格=<投入金額の場合->9.
    価格=>投入金額の場合->「金額が足りません」と表示->7.
9.  商品排出
10. お釣りを返す
11. 在庫を1減らす
12. 購入完了
13.1に戻る
```

### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

#### 機能1: 商品一覧表示

```
商品一覧表示：
商品データを先頭から順番に確認する ０から、i++
商品ID、商品名、価格、在庫数を表示する
在庫数が0の場合は「売り切れ」と表示する
```

#### 機能2: 投入金額表示

```
投入金額表示

1.初期段階の投入金額を表示する(この段階では投入されていないので0と表示)
2.ユーザーからの金額の入力を受け取る
3.入力された金額を表示する例:1000円
正の整数以外が入力された場合は2に戻る(正しい値を入力してくださいを表示する)
```

#### 機能3: 商品購入

```
F04 商品購入

1.  ユーザーに購入する商品の番号（ID)を入力させる
2.  商品番号が正しいか確認
3． 正しい場合->4.
    正しくない場合->「商品番号が間違っています」と表示->1.
4.  在庫があるか確認
5.  ある場合->6．
    ない場合->「売り切れ」と表示->1.
6.  商品名、価格を表示
7.  投入金額を入力
8.  価格=<投入金額の場合->9.
    価格=>投入金額の場合->「金額が足りません」と表示->7.
9.  商品排出
10. お釣りを返す
11. 在庫を1減らす
12. 購入完了
```

#### 機能4: お釣りの管理

```
1.投入金額-選択した商品の値段=お釣り
2.投入金額-選択した商品の値段が0以上であればお釣りの排出を行う
```

#### 機能5: 購入履歴の表示

```
購入履歴：
1.ユーザーが商品を購入する
2.購入した商品情報を取得する
・商品名
・価格
・購入日期
3.CSVファイルを開く
4.取得した購入データをCSV形式で追記する
5.ファイルを保存している閉じる
6.処理終了
```
